计算 (Computation) 与 通信 (Communication) 重叠原理解析

在异构加速器（如 GPU/TPU）或集群运算中，数据需要从网络/内存拉取到计算单元（通信耗时 $T_{comm}$），计算单元执行乘法累加（计算耗时 $T_{comp}$）。 如果串行执行，通道将产生大量空闲延迟。而利用双缓冲（Double Buffering）流水线，我们可以在计算当前块数据 $k$ 的同时，异步在后台拉取下一个数据块 $k+1$ 的数据。从而最大化隐蔽通信开销，提升总体硬件吞吐量！

流水线参数设置

矩阵切分块数 ($K$) 4 块

将运算切分成 $K$ 次块数据拉取与乘法累加

单块通讯耗时 ($T_{comm}$) 1.0 秒

从外部物理总线传输单块矩阵数据的时间

硬件计算核心进行局部块乘法的矩阵运算耗时

系统会自动配比 $T_{comm} \approx T_{comp}$ 实现极限重叠，逼近 0 空闲理论瓶颈。

高速缓存规范化体系 (Normalized Cache Architecture) 与分块寻址

内存墙（Memory Wall）是限制矩阵相乘效率的元凶。当进行 $C = A \times B$ 计算时，由于计算机读取内存数据是按高速缓存行 (Cache Line) 载入的，若循环寻址顺序不匹配（例如 B 矩阵按列跨步访问），会导致大量 L1 Cache Miss。 采用矩阵分块 (Tiling)，可让一个分块完全装进规范化的 Cache 结构内，使数据在被写回主存前得到最大程度的复用！

L1 缓存结构配置 (SRAM)

缓存总线行数 (Line Slots) 8 行
Cache Line 大小 (Elements) 4 浮点数/行

每次发生 Miss，将相邻的 N 个元素一同载入以提供空间局部性
